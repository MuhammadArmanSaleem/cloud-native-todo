# Todo App Overview - Phase II

## Purpose

A todo application that evolves from console app (Phase I) to full-stack web application (Phase II) with advanced features.

## Current Phase

**Phase II: Full-Stack Web Application**

## Tech Stack

- **Frontend**: Next.js 16+, TypeScript, Tailwind CSS
- **Backend**: FastAPI, SQLModel
- **Database**: Neon Serverless PostgreSQL
- **Auth**: Better Auth with JWT
- **Spec-Driven**: Claude Code + Spec-Kit Plus

## Features In Scope

### Basic Level (Core Essentials)
- [x] Task CRUD operations (web version)
- [x] User authentication (Better Auth)
- [x] Persistent storage (Neon PostgreSQL)

### Intermediate Level (Organization & Usability)
- [ ] Priorities & Tags/Categories
- [ ] Search & Filter
- [ ] Sort Tasks

### Advanced Level (Intelligent Features)
- [ ] Recurring Tasks
- [ ] Due Dates & Time Reminders

### Bonus Points
- [ ] Reusable Intelligence (Claude Code Skills)
- [ ] Cloud-Native Blueprints
- [ ] Multi-language Support (Urdu)
- [ ] Voice Commands

## Features Out of Scope

- Mobile app
- Real-time collaboration
- File attachments
- Task sharing between users

## Demo Flow

1. User signs up/logs in
2. User creates tasks with priorities and tags
3. User searches and filters tasks
4. User sets due dates and reminders
5. User creates recurring tasks
6. User uses voice commands (bonus)

## Acceptance Criteria

- All Basic Level features work as web application
- User authentication with Better Auth
- Data persists in Neon PostgreSQL
- All API endpoints secured with JWT
- Responsive frontend interface
- All Intermediate and Advanced features functional
