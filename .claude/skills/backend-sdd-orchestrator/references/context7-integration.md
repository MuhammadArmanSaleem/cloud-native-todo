# Context7 Integration for Backend Orchestration

How to use Context7 during backend SDD orchestration.

## When to Query Context7

### During Planning Phase

**Query for**:
- Latest FastAPI patterns
- Security best practices
- Database integration patterns
- Performance optimization

**Example**:
```text
Query: "FastAPI dependency injection, middleware, security authentication, database integration patterns"
Library: /websites/fastapi_tiangolo
```

### During Implementation

**Query for**:
- Specific API usage
- Edge case handling patterns
- Security patterns
- Error handling patterns

**Example**:
```text
Query: "FastAPI error handling, HTTPException, custom exception handlers"
Library: /websites/fastapi_tiangolo
```

### During Clarification

**Query for**:
- Resolve technical questions
- Find recommended approaches
- Understand trade-offs

**Example**:
```text
Query: "FastAPI Server Components vs Client Components, when to use async"
Library: /websites/fastapi_tiangolo
```

## Query Strategy

### Research Phase (sp.plan)

1. **Identify unknowns** from Technical Context
2. **Query Context7** for each unknown
3. **Consolidate findings** in research.md
4. **Resolve all NEEDS CLARIFICATION** items

### Implementation Phase

1. **Query before implementing** complex features
2. **Use latest patterns** from Context7
3. **Verify best practices** are followed
